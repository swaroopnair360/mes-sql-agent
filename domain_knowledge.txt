WIP_ORDER joins WIP_OPERATION using:
- WipOrderNo
- WipOrderType

WIP_OPERATION joins PROGRESS_STATUS using:
- ProgressStatus

While joining wip_operation, always use InsideProcessFlag = 1 condition

Use WIP_OPERATION.ProgressStatus for current status, not WIP_ORDER.

While joining RESOURCE_ table, always use both ResourceName & ResourceType
- ResourceType = 6 is machine

ResourceName is never the name of machine. Resource_.Name is the machine name always

Text_Translation helps in getting translation and description across all tables. By default used LanguageID as 1033 (English)

Order description needs to be found using textid from wip_order

All tables have CreatedBy, CreatedOn, LastUpdateOn, LastUpdatedBy columns

Wip order table stores both SAP Po and MES po. SAP PO is wipordertype 23 and MES PO is wipordertype 1

Meaning of labortype: 1 - Indirect, 2 - Job Direct, 3 - Job Downtime, 4 - Job Setup, 5 - Job Teardown, 6 - Planned Maintenance, 7 - Span (Shift), 8 - State (Event), 9 - Order, 10 - Product, 11 - Defect, 12 - Custom

Meaning of laborStatus: 1 - Open, 2 - Closed, 3 - Signed Off, 4 - Approved, 5 - Posted, 6 - Posted Signed Off, 7 - Posted
